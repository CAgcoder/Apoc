# APoc Demo 分镜 + 旁白脚本

**受众:** 大厂技术团队 leader / 初创公司 CTO。
**目标信号:** 不是"功能多",而是"这个人能想清楚并讲清楚架构决策"。每一段旁白都先说**为什么**,再说是什么。
**目标时长:** ~3 分钟(技术受众的甜区是 2–4 分钟)。
**语言:** 旁白中文(可转英文)。
**配音:** 推荐你本人录;脚本同时可直接喂 TTS 当后备。
**录制:** 方案 B——你用 ⌘⇧5 录屏(静音),我在浏览器里按下面的 choreography 控制节奏;旁白后期叠加。

---

## 录制前准备(我来做)

1. 后端 + 前端跑起来(`./run.sh` + `npm run dev`)。
2. 选定演示项目:`FinCore 企业级财务合规与供应链结算系统`(主题对技术受众够分量),终态演示用已批准的 `APOC 文档管理`。
3. 浏览器全屏、隐藏书签栏、用干净的 profile(无个人标签)。
4. 录制分辨率建议 1920×1080 或 1440p;窗口缩放调到内容清晰。

> **节奏约定:** 每个场景标了"画面停留时长"。你录制时按这个时长走,旁白后期对齐到同样的分段就能严丝合缝。

## 分镜表

|---|---|---|---|

## 旁白生成(后期)

录完静音视频后,旁白按场景分段生成、再合成:

**A. 真人录(推荐):** 照上面逐段念,每段单独录成 `seg1.wav … seg7.wav`,好对齐也好重录。

**B. 本地 TTS 后备:** 把每段文本生成音频(中文语音示例,质量一般,正式用建议先下 premium 语音):
```bash
say -v "Meijia" -o seg2.aiff "我描述需求,后端是一个 LangGraph 的 DAG 在跑……"
```

**C. 合成:** 把整条旁白音轨叠到录屏视频上:
```bash
# 多段旁白先拼成一条 narration.wav(按场景时长在段间留白),再合成:
ffmpeg -i screen.mov -i narration.wav \
  -c:v copy -c:a aac -map 0:v:0 -map 1:a:0 -shortest demo.mp4
```

**生成段加速(场景 2):** 先把那段单独剪出来加速,再拼回:
```bash
ffmpeg -i gen_segment.mov -filter:v "setpts=0.15*PTS" -an gen_fast.mp4   # ~6.6x
```
